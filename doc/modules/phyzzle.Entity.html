Module phyzzle.Entity

The Phyzzle physics entity.

Has a location, dimension, acceleration and various states.

Functions

new (t)
Create a new entity.

A table of overrides and initial state can be provided. x,y,w,h, static and collider can be overridden.

Parameters:

  • t Optional table of overrides and initial state.

Returns:

    Entity instance.
init ()
Initialize the map by creating any instance specific tables.

Called automatically by Entity:new.

Override in child entities to specify custom init functionality. All init functions will always be called unlike other overriden functions.

collision (e2, dx, dy)
Map collision response.

Called when this entity collides with another. The default collision response will touch the other entity. To pass over an entity override this function in a child table and remove the coollision response logic.

Parameters:

  • e2 The other entity.
  • dx Change in x.
  • dy Change in y.
move (dx, dy)
Map move response.

Called when this entity moves without colliding.

Parameters:

  • dx Change in x.
  • dy Change in y.
update (dt)
Update the entity.

For predictable results use a fixed timestep and then dt can be ignored. See the custom love.run function in main for a fixed time step game loop implemenation.

Parameters:

  • dt Change in time or delta time.
draw ()
Draw the entity.

The default draw function does nothing. Drawing can be implemented entirely in a separate renderer or the renderer can be implemented to call the entity's draw function.
